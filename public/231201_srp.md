# 概要
過去に行ったダメな設計をSOLID原則の単位責任の原則の観点から振り返る。

# 開発対象
開発対象はつぎのようなWindows Formアプリケーションだった。

## 要件
* Windows Formアプリケーション
* 言語: C#
* 無線で装置の温度、無線強度を受信する
* 無線接続状態（接続・未接続）を表示する
* 温度、無線強度をサードパーティ製のグラフ描画ライブラリで折れ線グラフを描画する
* 通信設定（装置ID、周波数帯の設定、無線伝送速度設定）
* セキュリティ設定（暗号化の有無、暗号キー）
* 装置設定（装置のLED色）
* 温度は摂氏・華氏で表示形式を変更可能
* グラフ表示更新間隔設定（100ms〜1000ms）
